## S.C.O.P.E. Framework

| Phase | Purpose | Activities |
|-------|---------|------------|
| **S**cope | Define boundaries, audience, objectives | Topic definition, audience profile, success criteria |
| **C**ollect | Gather context from existing sources | Load prior work, framework docs, community feedback |
| **O**rchestrate | Chain @agentQ modes strategically | Select pattern, configure mode sequence |
| **P**roduce | Generate deliverable artifacts | Execute modes, apply standards, embed philosophy |
| **E**xport | Package outputs, enable feedback loop | Format files, update registry, suggest sharing |

## Four Workflow Patterns

**Purpose:** Create training materials, workshops, tutorials

**Mode Sequence:**
1. --training (workshop materials)
2. --howto (step-by-step tutorials)
3. --help (reference documentation)

**Output:**
- `knowledge/{topic}/workshop.md` - Full workshop outline
- `knowledge/{topic}/tutorial.md` - Step-by-step guide
- `knowledge/{topic}/reference.md` - Quick reference

**Use Cases:**
- Onboarding new developers
- Internal training programs
- Community workshops
- Technical skill-building

**Extended Explanation:**

The /knowledge command orchestrates @agentQ's 13 content generation modes through the S.C.O.P.E. framework, embedding continuous learning where EXPORT outputs become future SCOPE inputs.

>

**Prerequisites:**
- Topic or framework name
- Optional: Context from /ideate, /research, or /analyze
- Output directory: `knowledge/` (created if missing)

>

**What It Does:**
1. SCOPE: Defines topic boundaries, identifies target audience, reviews prior sessions
2. COLLECT: Loads existing context from idea-context.md, research-context.md, framework docs
3. ORCHESTRATE: Chains @agentQ modes based on selected pattern (--teach, --publish, --share, --comprehensive)
4. PRODUCE: Generates content through orchestrated modes, applying Q101 standards and philosophy
5. EXPORT: Packages outputs, updates knowledge-registry.json, suggests sharing channels, enables feedback collection

>

**Output:**
- `knowledge/{topic}/` directory with generated content files
- `knowledge-registry.json` entry with session metadata and lineage tracking
- Sharing guide with distribution recommendations
- Feedback channels for continuous improvement

>

**Next Steps:**
1. /knowledge --teach "topic" (create educational content)
2. /knowledge --publish "topic" (create publication content)
3. /knowledge --share "topic" (create community content)

## Execution Steps

### STEP 0: Parse Arguments

Extract pattern, topic, and options:
```
/knowledge --teach "P.A.R.T.S. methodology"
  → pattern: --teach
  → topic: "P.A.R.T.S. methodology"
  → options: (none)

/knowledge --publish --idea=a7f3d821
  → pattern: --publish
  → topic: (from idea-context.md)
  → options: --idea=a7f3d821
```

### STEP 1: SCOPE Phase

**1.1 Define Topic Boundaries**

| Activity | Description | Output |
|----------|-------------|--------|
| Parse topic | Extract topic name | Topic string |
| Set scope limits | What's in/out of scope | Scope definition |
| Identify audience | Who will consume this? | Audience profile |
| Set objectives | What should they learn/do? | Success criteria |

**1.2 Review Prior Sessions**

Query `knowledge-registry.json` for prior work on this topic:
- Has this topic been covered before?
- What feedback was received?
- What gaps remain?

**1.3 Load Integration Context**

If `--idea={id}`, `--topic={id}`, or `--analysis={id}` provided:
- Load `idea-context.md` (from /ideate)
- Load `research-context.md` (from /research)
- Load `ANALYSIS-REPORT.md` (from /analyze)

**1.4 Select Pattern**

| Pattern | Use When | Output Types |
|---------|----------|--------------|
| --teach | Training/education needed | Workshop, tutorial, reference |
| --publish | Publication content needed | Book, whitepaper, catalog |
| --share | Community engagement needed | Social, quotes, stories |
| --comprehensive | Complete catalog needed | All 13 modes |

---

### STEP 2: COLLECT Phase

**2.1 Gather Framework Context**

Load relevant framework documentation:
- `frameworks/q101/*.md` - Methodology files
- `.claude/commands/*.md` - Command definitions
- `.claude/commands/agents/*.md` - Agent definitions
- `reference/q101/*.md` - Reference guides

**2.2 Import Community Feedback**

Check registry for feedback from prior sessions:
- What questions were asked?
- What confusion arose?
- What worked well?

**2.3 Identify Knowledge Gaps**

Compare existing content against scope objectives:
- What's missing?
- What needs clarification?
- What examples would help?

---

### STEP 3: ORCHESTRATE Phase

**3.1 Select Mode Sequence**

Based on pattern, configure @agentQ mode chain:

**--teach pattern:**
```
Sequence: --training → --howto → --help
Handoff: Workshop outline → Tutorial → Reference
```

**--publish pattern:**
```
Sequence: --book → --whitepaper → --index
Handoff: Chapter draft → Whitepaper → Catalog
```

**--share pattern:**
```
Sequence: --post → --quotes → --about
Handoff: Social thread → Wisdom → Story
```

**--comprehensive pattern:**
```
Sequence: All 13 modes in order
Handoff: Sequential with context preservation
```

**3.2 Configure Handoffs**

Define what context passes between modes:
- Topic definition
- Audience profile
- Prior outputs
- Framework context

**3.3 Set Quality Gates**

Define success criteria for each mode:
- Accuracy (matches Q101 principles)
- Completeness (covers all required aspects)
- Clarity (audience can understand)
- Actionability (readers can apply)

---

### STEP 4: PRODUCE Phase

**4.1 Execute Mode Sequence**

For each mode in the orchestration:

1. **Invoke @agentQ Mode**
   - Call `/agentQ --{mode} {topic}`
   - Pass context from previous mode
   - Apply mode-specific template

2. **Apply Q101 Standards**
   - Use consistent formatting
   - Include relevant examples
   - Embed "You got this!" philosophy
   - Link to reference documentation

3. **Create Teachable Moments**
   - Explain WHY, not just WHAT
   - Connect to broader principles
   - Provide multiple examples
   - Anticipate questions

4. **Validate Output**
   - Check against quality gates
   - Verify accuracy
   - Test clarity with internal review

**4.2 Format Content**

Ensure consistent structure:
- Clear headings
- Code examples (where appropriate)
- Visual diagrams (ASCII art)
- Cross-references to related content

---

### STEP 5: EXPORT Phase

**5.1 Package Outputs**

Create `knowledge/{topic}/` directory structure:

```
knowledge/
  {topic}/
    workshop.md           (--teach: training materials)
    tutorial.md           (--teach: step-by-step guide)
```

**5.4 Enable Feedback Loop**

The EXPORT phase completes the continuous learning cycle:
- Outputs are packaged for sharing
- Feedback channels are established
- Registry captures session for future SCOPE reference
- Next iteration ideas seed future sessions

**The Continuous Learning Loop:**
```
     ┌─────────────────────────────────────────┐
     │                                         │
     ▼                                         │
  SCOPE ──► COLLECT ──► ORCHESTRATE ──► PRODUCE ──► EXPORT
     ▲                                              │
     │                                              │
     └──────────── FEEDBACK LOOP ◄─────────────────┘
                   (Learn & Improve)
```

When someone uses the exported content, their feedback flows back into the registry, informing the SCOPE phase of future sessions.

## The Continuous Learning Philosophy

> "Knowledge shared is knowledge multiplied. Every lesson taught creates a teacher. Every insight shared sparks new ideas. This is continuous learning - where EXPORT becomes someone else's SCOPE."

### The Four Learning Principles

1. **Knowledge Multiplies When Shared**
   - Teaching reinforces your own understanding
   - Shared content reaches audiences you'd never reach alone
   - In /knowledge: PRODUCE creates teachable moments

2. **Feedback Fuels Improvement**
   - Community responses identify gaps
   - Questions reveal unclear explanations
   - In /knowledge: EXPORT enables feedback collection

3. **Every Expert Was Once a Learner**
   - Documentation creates new experts
   - Tutorials lower barriers to entry
   - In /knowledge: COLLECT builds on prior knowledge

4. **Wisdom Compounds**
   - Each session builds on previous ones
   - Registry captures institutional knowledge
   - In /knowledge: SCOPE learns from registry data
