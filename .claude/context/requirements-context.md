## Confirmed Requirements

### Functional Requirements

1. **Authentication & Authorization**
   - Email/password login with bcrypt hashing
   - JWT tokens stored in HttpOnly cookies (jose library)
   - Access token: 1h expiry, Refresh token: 7d expiry
   - 4 roles: Head Coach > Coach > Council Leader > Student
   - Role-based route protection via Next.js middleware

2. **3-Level Navigation (from prototype)**
   - L1: Batch Overview - council cards with progress stats (HC only)
   - L2: Council Detail - student list with goal progress buttons (HC + Coach)
   - L3: Student Detail - tabbed view with Attendance + Goals (all roles)
   - Role-based login destinations: HC→L1, Coach→L2, Student→L3(Goals tab)

3. **Goal Management (S.M.A.R.T.e.r. Framework)**
   - 3 goal types per student: Enrollment, Personal, Professional
   - Goal fields: statement, values, S.M.A.R.T.e.r. criteria
   - 12-week milestone tracking with cumulative percentages
   - Action items with checkboxes per week
   - Results/outcomes with checkboxes per week
   - Live progress auto-calculated from checked items

4. **Attendance Tracking**
   - Weekly meetings grid (12 weeks, status: Present/Late/Absent/No Data)
   - Coaching calls table (M-T-W-Th-F per week)
   - Intensives attendance (3 events, 2 days each)
   - Workshops attendance (2 events)
   - Paint mode for quick-set status
   - Consistency score calculation

5. **Council Management**
   - Create/edit/delete councils (HC only)
   - Assign coaches and council leaders
   - Council themes
   - Buddy system within councils

6. **Communication**
   - Coach-Student chat per student
   - AI Coach chat (rule-based MVP, future: Claude API)
   - Coach approvals workflow

7. **Notifications & Reminders**
   - In-app notifications (bell icon, unread count)
   - Email reminders for coaching calls
   - Weekly check-in reminders (Monday)
   - Low progress alerts
   - Milestone deadline reminders

8. **Export & Reporting**
   - PDF progress reports (student, council, batch levels)
   - CSV data exports
   - Dashboard charts (Recharts)

9. **Offline/PWA**
   - Service Worker with cache-first for static, network-first for API
   - IndexedDB for offline goal updates
   - Background sync on reconnection
   - Offline status badge + pending changes counter
   - Installable PWA

10. **Dark Mode**
    - Toggle dark/light theme
    - Persist preference

## Web Research Findings

### Critical: SQLite + Vercel Limitation
- Vercel uses ephemeral serverless filesystem; local SQLite files don't persist
- **Solution:** Turso (hosted libSQL) - free tier, Vercel marketplace integration
- Local SQLite for development, Turso for production
- Drizzle ORM has first-class Turso support via `@libsql/client`

### Authentication Best Practices
- bcrypt with 10 salt rounds for password hashing
- jose library (HS256) for JWT - edge-compatible
- Store JWT in HttpOnly, Secure, SameSite=Strict cookies
- Minimal JWT payload: user ID + role only
- Access token 1h, refresh token 7d
- Next.js middleware for route protection

### PWA/Offline Architecture
- Cache-first for static assets (install-time precache)
- Network-first with cache fallback for API responses
- IndexedDB for offline goal update queue
- Background Sync API for automatic sync on reconnect
- Serwist or Workbox 7 for service worker management

### shadcn/ui Dashboard Patterns
- Card for KPIs, DataTable for student lists, Tabs for L3 views
- Sidebar (persistent desktop) + Sheet (mobile drawer)
- Design tokens for consistent theming
- Recharts for data visualization
